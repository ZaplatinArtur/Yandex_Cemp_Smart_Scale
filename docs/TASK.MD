# Пайплайн распознавания весовых товаров

## Описание проекта
Данный проект представляет собой высокоточную систему компьютерного зрения для умных весов (касс самообслуживания). Система автоматически распознает тип и сорт весового товара (фрукты и овощи), положенного на платформу, устойчива к перекрытиям и оптическим иллюзиям, а также имеет встроенную защиту от мошенничества.

Главная архитектурная особенность — использование **Metric Learning и векторных баз данных (K-NN)**. Это делает систему **Zero-Shot**: для добавления нового сорта товара в ассортимент магазина не требуется переобучение нейросетей, достаточно добавить эталонный вектор в базу данных.

## Архитектура пайплайна

Процесс обработки каждого кадра с камеры весов разделен на 5 изолированных этапов:

### 1. Детекция аномалий
* **Технология:** `MediaPipe Hands`
* **Логика:** Прежде чем тратить вычислительные ресурсы на тяжелые модели, кадр проверяется на наличие кистей рук. Если покупатель придерживает пакет руками или опирается на весы, система блокирует инференс и выдает предупреждение, предотвращая некорректное сканирование.

### 2. Object Detection & Cropping (Локализация)
* **Технология:** `Ultralytics YOLO11n`
* **Логика:** Детектор находит все объекты на платформе весов. Для отсечения лишнего фона весов выбирается объект с максимальной уверенностью (`Confidence Score`). Попиксельной сегментационной маской (`yolo11-seg`)

### 3. Построение эмбеддингов
* **Технология:** `Meta DINOv2`
* **Логика:** Вырезанное изображение продукта подается в фундаментальную модель DINOv2 (`facebook/dinov2-small`), предварительно переведенную в режим инференса. Модель генерирует семантический вектор (эмбеддинг) размерностью 384 признака. Вектор проходит обязательную $L_2$-нормализацию.

### 4. K-NN
* **Технология:** `pgvector`
* **Логика:** Полученный нормализованный вектор отправляется в базу данных. Векторная БД выступает как непараметрический классификатор: она возвращает ТОП-K ближайших эталонных векторов из каталога вместе с их метаданными (название, цена за грамм, ID).

### 5. API Layer
* **Технология:** `FastAPI`, `Pydantic`
* **Логика:** Сервер принимает изображения с весов в асинхронном режиме, оркестрирует весь инференс моделей и возвращает на кассу строгий JSON-ответ с названием товара, уверенностью предсказания и рассчитанной ценой.

## Технологический стек
* **Computer Vision:** `OpenCV`, `PyTorch`, `Transformers`, `Ultralytics`
* **Models:** `YOLO11`, `MediaPipe`, `DINOv2`
* **Backend & MLOps:** `FastAPI`, `PostgreSQL + pgvector`
* **Math:** `NumPy`, `Scikit-learn`

## Итоговый пайплайн:
1. ожидание стабилизации веса
2. снятие фото
3. отправка фото и веса на сервер
4. детекция рук - если есть, то отправляем предупреждение
5. если рук нет, то распознаём продукт, делаем обработку фото вырезки
6. переводим полученное изображение в вектор
7. поиск в БД по ближайшим
8. выбор конкретного типа и сорта продукта
9. обработка и вывод ответа
